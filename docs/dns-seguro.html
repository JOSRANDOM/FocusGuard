Por qué un bloqueo a veces no funciona — y cómo arreglarlo

Guía rápida para desactivar "DNS seguro" en tu navegador, el ajuste que puede hacer que FocusGuard parezca fallar aunque esté funcionando correctamente.

FocusGuard · Guía de usuario

1El síntoma

Esto es exactamente lo que pasa cuando tu navegador usa DNS seguro.

🚫
Bloqueas la app
Activas el switch de una plataforma (ej. YouTube) en FocusGuard.
→
📡
Error de conexión
Al entrar, ves "Conéctate a Internet" — el bloqueo funcionó.
→
🔄
Se recarga sola
A los pocos segundos, la página reintenta.
→
▶️
Carga normal
El sitio abre como si no hubiera bloqueo.
ℹ️
No es un error de FocusGuard

El archivo hosts del sistema sigue bloqueado todo el tiempo — lo verificamos escribiéndolo y revisándolo varios segundos después, sin cambios. Es el navegador el que decide ignorarlo.

2La causa real

"DNS seguro" (DNS-over-HTTPS) hace que el navegador le pregunte a un servidor externo cómo resolver un dominio, en vez de preguntarle al sistema operativo — que es quien respeta el archivo hosts donde FocusGuard escribe los bloqueos.

⚠️
Le pasa a cualquier bloqueador basado en "hosts"

Cold Turkey, Freedom y herramientas similares tienen la misma limitación. No hay forma de arreglarlo solo desde la app — la solución vive en la configuración del navegador.

3Desactivar DNS seguro por navegador

Sigue los pasos del navegador que uses. Toma menos de un minuto.

🌐

Google Chrome

chrome://settings/security
  • 1Abre el menú ⋮ → Configuración → Privacidad y seguridad → Seguridad.
  • 2Busca la sección "Usar DNS seguro".
  • 3Desactiva el interruptor (o elige "Con tu proveedor de servicios actual" como alternativa más permisiva).
Usar DNS seguro
OFF
🧭

Microsoft Edge

edge://settings/privacy
  • 1Ve a Configuración → Privacidad, búsqueda y servicios.
  • 2Baja hasta "Servicios de seguridad".
  • 3Desactiva "Usar DNS seguro para especificar cómo se resuelven las búsquedas de DNS".
Usar DNS seguro
OFF
🦊

Mozilla Firefox

about:preferences#general
  • 1Baja hasta "Configuración de red" → botón "Configuración".
  • 2Desmarca "Habilitar DNS mediante HTTPS" (o elige "Desactivado").
DNS mediante HTTPS
OFF

4Otros casos

🧭

Brave, Opera y otros basados en Chromium

Misma opción, normalmente en Configuración → Privacidad y seguridad → Seguridad → DNS seguro — los pasos son equivalentes a los de Chrome.

🍎

Safari (macOS)

Safari usa el resolutor de DNS del sistema por defecto y normalmente no necesita este ajuste. Si el bloqueo no funciona ahí, revisar en su lugar Preferencias del Sistema → Red → DNS por si hay un servidor DNS-over-HTTPS configurado a nivel de sistema.

✅
Después de desactivarlo

Cierra por completo el navegador y ábrelo de nuevo. El bloqueo de FocusGuard debería mantenerse estable, sin recargarse hacia el sitio real.
